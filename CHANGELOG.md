# 更新日志

## 2026-03-09 - 修复用户管理界面的中英文切换问题

### 问题描述
- 用户管理界面的所有文字都是硬编码的中文，没有使用 i18n 翻译函数
- 切换语言后，用户管理界面的文字不会变化

### 解决方案
1. **更新 i18n 翻译文件**
   - 在 `src/i18n/zh-CN.json` 中添加了 `admin.users` 的完整翻译键
   - 在 `src/i18n/en-US.json` 中添加了对应的英文翻译
   - 修复了 JSON 格式问题

2. **修改 UsersManagement.vue**
   - 导入 `useI18n` 并使用 `t()` 函数
   - 将所有硬编码的中文替换为 `$t()` 翻译函数
   - 包括：页面标题、按钮文字、搜索框、表头、弹窗、提示信息等
   - 所有动态消息也使用翻译函数

### 修改的文件
- `src/i18n/zh-CN.json` - 添加用户管理界面中文翻译键，修复 JSON 格式
- `src/i18n/en-US.json` - 添加用户管理界面英文翻译键
- `src/views/admin/UsersManagement.vue` - 替换硬编码中文为翻译函数

### 测试建议
1. 刷新页面，检查默认语言是否正确显示
2. 点击 English 按钮，检查用户管理界面的所有文字是否切换为英文
3. 点击 中文 按钮，检查所有文字是否切换回中文
4. 测试添加用户弹窗的所有文字
5. 测试编辑用户弹窗的所有文字
6. 测试查看用户详情弹窗的所有文字
7. 测试删除/封禁用户的确认提示
8. 测试搜索框和筛选器的文字
9. 测试分页按钮的文字

## 2026-03-09 - 修复管理控制台中英文切换问题

### 问题描述
- 点击 English 按钮后，虽然语言状态已切换到 en-US，但界面文字仍然显示中文
- AdminDashboard.vue 和 AdminNavbar.vue 中的文字都是硬编码的中文，没有使用 i18n 翻译函数

### 解决方案
1. **更新 i18n 翻译文件**
   - 在 `src/i18n/zh-CN.json` 中添加了 `admin.navbar` 和 `admin.dashboard` 的翻译键
   - 在 `src/i18n/en-US.json` 中添加了对应的英文翻译

2. **修改 AdminNavbar.vue**
   - 将所有硬编码的中文替换为 `$t()` 翻译函数
   - 包括：导航栏标题、菜单项、按钮文字、退出登录确认框等

3. **修改 AdminDashboard.vue**
   - 导入 `useI18n` 并使用 `t()` 函数
   - 将所有硬编码的中文替换为 `$t()` 翻译函数
   - 包括：页面标题、数据卡片标题、图表标题、功能导航按钮等
   - 图表标题也使用翻译函数，确保切换语言时图表标题也会更新

### 修改的文件
- `src/i18n/zh-CN.json` - 添加中文翻译键
- `src/i18n/en-US.json` - 添加英文翻译键
- `src/components/AdminNavbar.vue` - 替换硬编码中文为翻译函数
- `src/views/admin/AdminDashboard.vue` - 替换硬编码中文为翻译函数

### 测试建议
1. 刷新页面，检查默认语言是否正确显示
2. 点击 English 按钮，检查所有文字是否切换为英文
3. 点击 中文 按钮，检查所有文字是否切换回中文
4. 检查图表标题是否随语言切换而更新
5. 刷新页面，检查语言设置是否保持（localStorage）
